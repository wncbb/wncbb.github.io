Russ Cox’s blog ablout golang interface

如上图所示。interface分为两部分:

  1. itable: 第一部分为原始数据的type信息，第二部分为data符合该interface的函数列表(b的其他方法不会出现)。

  2. data: 指向原始的数据。

只有这两部分都是nil的时候，才算做nil

package main

type A interface{}

type B interface {
	Func()
}

type SB struct {
}

func (SB) Func() {}

func main() {
    // a is nil
	var a A
	if a == nil {
		println("a is nil")
	} else {
		println("a is not nil")
	}

    // b is nil
	var b B
	if b == nil {
		println("b is nil")
	} else {
		println("b is not nil")
	}

    // b is not nil, 因为b装载的data是nil
    // 但是itable却不是nil(sb是有类型的，为SB)， 
    // 因此不是nil
    // go2.0不知道会不会改
	var sb *SB
	b = sb
	if b == nil {
		println("b is nil")
	} else {
		println("b is not nil")
	}
}
